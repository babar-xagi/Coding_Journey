# 📜 Learning Rules & Habits

To keep this programming journey focused, productive, and consistent, I commit to following these baseline guidelines.

---

## 🕒 Language Focus Ratios & Routine

1. 🐍 **Python Daily First**
   - Python is the primary career target (70% focus).
   - Begin every daily study block with Python (minimum 30–45 mins).
   - Target AI/ML libraries, FastAPI configurations, and professional package design.

2. 🦀 **Rust 3–4 Days Per Week**
   - Rust represents the systems/performance target (20–25% focus).
   - Dedicate at least 3 sessions per week to Rust.
   - Focus heavily on ownership rules, memory models, safety checks, and Python bindings (PyO3).

3. 📘 **TypeScript Only for Practical/Frontend Use**
   - TypeScript is the UI target (5–10% focus).
   - Limit scope to building interactive frontend UI components and API consumers.
   - Do not spend time on node backends or non-browser tooling unless directly related to a project dashboard.

---

## 📝 Learning & Practice Rules

* **Concept Rules**: Every new concept folder must contain:
  1. Detailed revision notes in the README file.
  2. Executable code demonstrating the concept.
  3. A mini practical exercise solving a small logic problem.
* **Weekly Revision Routine**: Set aside one day per week (e.g., Saturday) as a **Revision & Logging Day**. Do not learn new syntax; instead, review completed concepts, fill in gaps, and practice refactoring.
* **Monthly Milestones**: Complete at least one mini-project every month from [PROJECTS.md](./PROJECTS.md) and document the result.
* **No Side Quests**: Do not add, learn, or experiment with other programming languages (e.g., Go, C, C++, Java, PHP) inside this repository. Keep the layout focused only on Python, Rust, and TypeScript.
* **Repository Cleanliness**: Keep the structure clean. Do not commit temporary build binaries (`target/`, `venv/`, `node_modules/`, `.pytest_cache/`). Ensure a thorough `.gitignore` is in place.

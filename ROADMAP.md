# 🗺️ Learning Roadmap

This document outlines the detailed roadmap divided into ten distinct phases. Each phase targets specific concepts to be learned concurrently in Python, Rust, and TypeScript, showing clear milestones, tasks, and expected outcomes.

---

## Phase 0: Repository Setup
* **Focus**: Workspace, tooling, and workflow automation.
* **Tasks**:
  - [x] Create directory structure
  - [x] Setup issue templates and Markdown templates
  - [x] Author core documentation files (`README.md`, `ROADMAP.md`, `PROGRESS.md`, etc.)
  - [x] Test relative linking within GitHub interface
* **Expected Outcome**: A functional, highly navigable learning repository with clear workflows.
* **Status**: 🟢 Completed

---

## Phase 1: Programming Foundations
* **Focus**: Variable declaration, mutability, primitive types, and basic tooling.
* **Tasks**:
  - [ ] Write Hello World in all three languages
  - [ ] Compare mutable vs immutable declarations
  - [ ] Document type systems (dynamic vs static/nominal vs structural)
  - [ ] Complete variables/data-types practice exercises
* **Expected Outcome**: Understanding how memory, values, and names interact in Python, Rust, and TypeScript.
* **Status**: 🟡 In Progress

---

## Phase 2: Control Flow + Functions
* **Focus**: Conditionals, branching, loops, functions, and scopes.
* **Tasks**:
  - [ ] Implement conditional structures (`if-else`, pattern matching, switch statements)
  - [ ] Implement loop patterns (`for`, `while`, loop expression, iterator patterns)
  - [ ] Define functions: parameters, return types, generic variables, anonymous functions/closures
* **Expected Outcome**: Mastery over execution flows and functional decomposition in all three environments.
* **Status**: 🔴 Not Started

---

## Phase 3: Data Structures
* **Focus**: Collections (lists, arrays, vectors, dicts, hash maps, tuples, sets).
* **Tasks**:
  - [ ] Compare Python Lists/Tuples vs Rust Vectors/Arrays vs TS Arrays/Tuples
  - [ ] Implement associative mappings (Python Dict, Rust HashMap, TS Map/Object)
  - [ ] Analyze memory layouts (heap vs stack, dynamic resizing)
* **Expected Outcome**: Choosing and utilizing efficient data structures across dynamic and static systems.
* **Status**: 🔴 Not Started

---

## Phase 4: OOP / Structs / Objects
* **Focus**: Code organization models (classes vs structs vs object types).
* **Tasks**:
  - [ ] Define Python classes (methods, inheritance, dunder methods)
  - [ ] Define Rust structs and enums (with `impl` blocks for methods)
  - [ ] Define TypeScript custom interfaces, classes, and types
* **Expected Outcome**: Ability to model domain-specific data structures utilizing OOP and composition.
* **Status**: 🔴 Not Started

---

## Phase 5: Files, Errors, Modules
* **Focus**: IO operations, error propagation/handling, and module layout.
* **Tasks**:
  - [ ] Read and write files synchronously and asynchronously
  - [ ] Study error patterns (try-catch, Result/Option Monads, Panic, Exceptions)
  - [ ] Setup multi-file modules, imports, exports, and crates
* **Expected Outcome**: Safe application patterns that handle IO failures gracefully.
* **Status**: 🔴 Not Started

---

## Phase 6: Backend + APIs
* **Focus**: Web servers, endpoint definition, and network requests.
* **Tasks**:
  - [ ] Setup FastAPI web server in Python
  - [ ] Build a lightweight Actix-web/Axum server in Rust
  - [ ] Create API routes in Express/Next.js using TypeScript
* **Expected Outcome**: Full-stack capability to design, serve, and query HTTP API endpoints.
* **Status**: 🔴 Not Started

---

## Phase 7: AI/Data Science with Python
* **Focus**: Deep-dive into Python's primary domain.
* **Tasks**:
  - [ ] Work with NumPy and Pandas for data manipulation
  - [ ] Train a basic model with Scikit-learn
  - [ ] Implement a retrieval pipeline using basic LLM APIs
* **Expected Outcome**: Proficient ingestion, parsing, and ML inference operations in Python.
* **Status**: 🔴 Not Started

---

## Phase 8: Rust Performance + Python Extensions
* **Focus**: Bridging the performance gap between Python and Rust.
* **Tasks**:
  - [ ] Study low-level concurrency and memory safety in Rust
  - [ ] Write a compute-heavy utility in Rust (e.g., parsing, math)
  - [ ] Expose the Rust code to Python using PyO3 / Maturin
* **Expected Outcome**: Building high-speed python packages written in native Rust.
* **Status**: 🔴 Not Started

---

## Phase 9: Integrated Real-World Projects
* **Focus**: Stitching the three languages together in real systems.
* **Tasks**:
  - [ ] Develop a full application with Rust core engine, Python ML process, and TS Frontend
  - [ ] Deploy utilizing containerized workflows (Docker, Compose)
  - [ ] Maintain logs, tests, and CI/CD pipelines
* **Expected Outcome**: Production-grade microservices system showing off the unique benefits of all three languages.
* **Status**: 🔴 Not Started
